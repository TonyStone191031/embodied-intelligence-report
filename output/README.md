# output 目录说明

## 用途

`output/` 用于存放可再生成的导出产物与检查报告，例如：

1. 结构化清单校验结果
2. 合并版 markdown
3. PDF / DOCX 导出中间产物
4. 批量制图输出

当报告版本进入冻结与发布阶段时，推荐使用 `output/exports/<version>/` 作为正式导出产物目录。

## 当前文件预期

当前阶段优先存放：

1. 图表资产清单校验报告
2. 后续脚本生成的图表或统计结果
3. 版本冻结后导出的 `.docx` / `.pdf` / 合并稿

## 维护原则

1. `output/` 中的文件默认视为可再生成产物，而不是长期编辑源。
2. 若导出内容有误，应修改 `docs/report/`、相关资产或 `code/export/` 脚本后重新生成。
3. 不建议直接手工维护导出的 Word / PDF，并把它们回写成报告源文件。
