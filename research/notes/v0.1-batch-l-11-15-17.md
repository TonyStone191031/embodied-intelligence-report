## 2. 新增关键资料

### 2.1 VLA、动作表示与训练路线

1. OpenVLA（arXiv:2406.09246）
   - 关键信息：7B 开源 VLA；约 `970k` 真机示教；Llama 2 + DINOv2 + SigLIP；强调可微调与开源协议价值。
   - 对应写作点：
     - `55.2` 继续作为“协议级开放”代表
     - `56.2/56.4` 用来说明“统一模型”并不等于吞掉全部技能层
2. GR00T N1（arXiv:2503.14734）
   - 关键信息：双系统架构；`System 2` 解释视觉/语言，`System 1` 生成实时动作；面向 humanoid foundation model。
   - 对应写作点：
     - `55.3` 继续强化“平台级路线”而非单模型路线
     - `56.2/56.4` 用来解释“基础模型 + 执行层”责任分工
3. Figure Helix（Figure 官方，2025-02-20）
   - 关键信息：`S2` 在 `7-9 Hz` 运行，`S1` 在 `200 Hz` 运行；双嵌入式低功耗 GPU；连续高维动作；强调 whole upper body control。
   - 对应写作点：
     - `55.5` 作为企业路线如何切分高低频责任的公开样本
     - `73.2`、`75.2` 用来解释异步推理与分层实时控制
4. Gemini Robotics（Google DeepMind 官方，2025-03-12）
   - 关键信息：把 Gemini 2.0 多模态推理下探到 VLA / ER 路线；强调“通用多模态底座 -> 机器人动作接口”。
   - 对应写作点：
     - `55.4` 补强“通用多模态底座向机器人下探”的谱系判断
5. Gemini Robotics On-Device（Google DeepMind 官方，2025-06-24）
   - 关键信息：本地低时延；脱网可运行；支持少量示教快速适配；面向 ALOHA / Franka / Apollo 等不同本体。
   - 对应写作点：
     - `55.4` 说明闭源路线也在主动回答部署责任
     - `75.2`、`83.2/83.4` 说明端侧 VLA 不是单纯压模型，而是重写频率分工
6. SmolVLA（arXiv:2506.01844）
   - 关键信息：可单 GPU 训练；可部署在 consumer GPU 甚至 CPU；引入异步推理栈与 chunked action generation。
   - 对应写作点：
     - `83.2/83.4` 作为“小模型 + 异步推理 + 社区平台”代表
     - `75.2` 作为边缘推理重构而非简单压缩的证据
7. Riemannian Flow Matching Policy（arXiv:2403.10672）
   - 关键信息：强调多模态动作分布与更低推理代价；证明 flow matching 在机器人策略中的可行性。
   - 对应写作点：
     - `53.3` 用来给 flow matching 更明确的机器人学入口
8. Action-to-Action Flow Matching（arXiv:2602.07322）
   - 关键信息：用历史 proprioceptive action 代替随机噪声起点；单步推理可到 `0.56 ms`；强调速度、鲁棒性与泛化。
   - 对应写作点：
     - `53.3` 用来说明 2026 年的 flow matching 路线已开始围绕“推理时延”直接重写采样起点
9. Flowing With Purpose / LAFM（arXiv:2606.23420）
   - 关键信息：以潜在动作先验替代固定高斯；在真实部署与 LIBERO-90 上给出收益；强调 action space heteroscedasticity。
   - 对应写作点：
     - `53.3` 用来说明 flow matching 已不只是“更快的 diffusion 替代品”，而是在重写动作先验结构

### 2.2 硬件、部署与系统软件栈

1. Jetson Orin 官方页面
   - 关键信息：同一代模块覆盖 `67 TOPS-275 TOPS`；功耗区间大致 `7W-60W`；JetPack + Isaac ROS + Platform Services 共同构成软件栈。
   - 对应写作点：
     - `72.2` 强化“同叫端侧 GPU，预算完全不同”
     - `72.3`、`75.2` 强化“模块、运行时、中间件一起设计”
2. ROS 2 官方仓库
   - 关键信息：ROS 2 被维护为一组围绕节点通信、工具链、中间件适配共同演化的软件集合，而非单一消息库。
   - 对应写作点：
     - `73.2`、`73.4` 说明 ROS / ROS 2 的价值在于定义系统边界与可替换接口，而不是“能发消息”

### 2.3 shared autonomy、端侧与持续自主

1. Human-Robot Mutual Adaptation in Shared Autonomy（arXiv:1701.07851）
   - 关键信息：shared autonomy 不是“人类永远正确”；系统需同时估计目标、适应性与信任边界。
   - 对应写作点：
     - `82.3` 说明 shared autonomy 的经典问题设定是“共同闭环”而非临时接管
2. Human-Centered Shared Autonomy for Motor Planning, Learning, and Control Applications（arXiv:2506.16044）
   - 关键信息：2025 视角下 shared autonomy 仍被视为长期范式；强调人类意图、控制混合与伦理边界。
   - 对应写作点：
     - `82.2/82.3/82.4` 用来更新 shared autonomy 的现代口径
3. Preference-Calibrated Human-in-the-Loop Reinforcement Learning for Robotic Manipulation（arXiv:2606.03949）
   - 关键信息：利用人类干预诱发的隐式偏好做 credit reassignment；在真机任务上提升成功率与收敛速度。
   - 对应写作点：
     - `82.2` 说明偏好不只是“二选一标注”，而是可以进入 credit assignment
4. Freeform Preference Learning for Robotic Manipulation（arXiv:2606.32027）
   - 关键信息：允许用自然语言偏好轴定义速度/安全/仔细程度等行为维度；强调 reward-conditioned policy。
   - 对应写作点：
     - `82.2` 说明 preference learning 已从二元偏好扩展到多维行为轴
5. RoboCopilot（arXiv:2503.07771）
   - 关键信息：无缝切换人类和自主策略；双臂任务中把 interactive imitation learning 变成连续教学过程。
   - 对应写作点：
     - `82.2/82.3` 说明“示教 - 纠偏 - 共融控制”可以合并成一条交互式学习链
6. Towards Long-term Autonomy: A Perspective from Robot Learning（arXiv:2212.12798）
   - 关键信息：长时自主的关键在在线学习、环境变化适应、板载计算资源约束。
   - 对应写作点：
     - `84.2-84.4` 继续把持续自主压回基础设施问题
7. DexFlyWheel（arXiv:2509.23829）
   - 关键信息：IL + residual RL + rollout + data augmentation 的闭环数据飞轮。
   - 对应写作点：
     - `84.3` 说明自动数据回流不是“全量回灌”，而是有选择的闭环增广
8. RADAR（arXiv:2603.11811）
   - 关键信息：语义规划 + 自动成功评估 + 自主环境 reset + 数据路由，把采集做成闭环引擎。
   - 对应写作点：
     - `84.3/84.4` 说明持续自主高度依赖“自动采集 + 自动复位 + 自动筛选”
9. Robot-Powered Data Flywheels（arXiv:2511.19647）
   - 关键信息：部署机器人收集在野数据，并回流适配基础模型；强调 deployment as data engine。
   - 对应写作点：
     - `84.3/84.4` 说明持续自主不是单机器人自我修炼，而是“部署 -> 采集 -> 适配 -> 回放”的组织闭环
