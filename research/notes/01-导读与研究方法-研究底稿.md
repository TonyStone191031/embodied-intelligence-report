# 01 导读与研究方法 - 研究底稿

## 1. 章节定位

本章不是普通导言，而是整份报告的方法论入口。

它至少承担五个作用：

1. 说明这份报告究竟试图解决什么问题，而不是只介绍“具身智能是什么”
2. 说明这份报告为什么不能按“新闻汇总”或“单篇论文综述”方式来写
3. 给出后续所有章节都要遵守的证据分层与判断规则
4. 解释整份报告的结构设计逻辑
5. 为后续版本维护建立统一的增量更新方法

## 2. 本章核心研究问题

### 2.1 主问题

如何把《具身智能行业和技术发展调研报告》的第一部分写成一章真正可用于长期学习与研究的方法论导论，而不是普通概述？

### 2.2 子问题

1. 这份报告的研究对象、读者假设和使用场景是什么？
2. 为什么这份报告必须区分“教材式基础”“技术综述”“产业分析”和“长期跟踪框架”？
3. 在具身智能这种跨学科、高噪声、高叙事密度领域，应如何划分资料类型与证据等级？
4. 为什么报告必须采用版本化维护，而不能把它当作一次性长文？
5. 本章应如何与后文的定义、主线回顾、分类体系和技术章节形成接口？

## 3. 研究与写作边界

### 3.1 本章应覆盖

1. 报告目标与读者定位
2. 研究范围与明确边界
3. 证据来源与证据等级
4. 整体结构设计逻辑
5. 版本维护与长期更新机制

### 3.2 本章不应过度展开

1. 具体技术路线细节
2. 企业竞争格局
3. 单一模型或单篇论文的深入评测
4. 具体 benchmark 的细节比较

这些内容应在后续章节展开，本章只负责建立方法框架与阅读坐标。

## 4. 拟采用的证据与参考类型

### 4.1 方法论与证据治理参考

1. PRISMA 2020
   - 用途：支持“系统综述应强调检索、筛选、报告透明性”的写法
   - 备注：本报告不是医学系统综述，但其“透明地说明检索与纳入规则”的精神可借用
2. Cochrane Handbook current version
   - 用途：支持“范围界定、纳入标准、检索、偏差考虑、综合与解释”这一研究工作流
3. Oxford CEBM Levels of Evidence
   - 用途：支持“不同来源的证据强弱不同，应显式分层”的写法

### 4.2 具身智能领域的一手材料类型

1. 论文与技术报告
2. 官方项目主页
3. GitHub 代码库
4. benchmark 与数据集主页
5. 企业官网、官方博客、发布会材料

### 4.3 本章的关键判断

1. 本报告不是单次输出，而是长期维护的研究资产
2. 本报告的方法论要求比普通科普文更接近“研究综述 + 产业研究 + 版本化知识库”
3. 对具身智能这类叙事密集领域，必须把事实、推断、评价显式分离
4. 报告第一部分本身就需要足够严谨，否则后续章节会失去统一标准

## 5. 可直接用于正文的来源摘记

### 5.1 Oxford CEBM

来源：
<https://www.cebm.ox.ac.uk/resources/levels-of-evidence/oxford-centre-for-evidence-based-medicine-levels-of-evidence-march-2009>

可用信息：

1. CEBM 明确给出了不同问题类型的 evidence levels
2. 系统综述与更高质量研究通常位于更高证据层级
3. 专家意见与第一性原理位于更低层级

可用于正文的作用：

1. 论证“不同来源不能被同权对待”
2. 说明本报告为何要区分论文、官方资料、媒体评论和市场叙事

### 5.2 Cochrane Handbook current version

来源：
<https://www.cochrane.org/authors/handbooks-and-manuals/handbook/current>

可用信息：

1. 当前版本为 Version 6.5, 2024
2. 目录显式包含 scope/questions、inclusion criteria、searching/selecting、bias、synthesis、interpreting results 等环节

可用于正文的作用：

1. 说明严肃研究写作不是“先写结论”，而是有明确工作流
2. 支撑本项目采用“先研究、后写作、再审查”的章节流程

### 5.3 PRISMA 2020

来源：
BMJ 论文页受限，当前可暂通过公开二手索引与通行常识引用其核心作用；后续如果能获得开放正文，再补更直接的一手引用。

当前可用结论：

1. PRISMA 2020 的核心价值在于提高系统综述报告的透明性
2. 对本项目最重要的启发不是医学细节，而是“检索、筛选、纳入和报告规则应明示”

## 6. 建议回写到正文的强化点

### 6.1 第一节要加强的点

1. 报告不是“知道热点”，而是“建立判断坐标系”
2. 读者具备控制/机器人背景，因此本报告要偏研究型而非入门型
3. 报告的产出形式应被定义为“长期维护的研究底稿”

### 6.2 第二节要加强的点

1. 边界必须写得更硬，避免后文不断概念漂移
2. 明确写出为什么自动驾驶、纯软件智能体与通用多模态应用不能直接等同于本报告的主对象

### 6.3 第三节要加强的点

1. 不能只写“论文优先”，还要写为什么
2. 要显式给出证据层级思想
3. 要说明演示视频、企业宣传和媒体报道的典型风险

### 6.4 第四节要加强的点

1. 不是只列目录，而要解释结构的必要性
2. 明确基础层、技术层、产业层、判断层之间的逻辑依赖

### 6.5 第五节要加强的点

1. 说明版本更新不仅是补信息，更是修正判断
2. 说明为什么本项目采用目录版本化
3. 说明“current/ + versions/”的研究意义

## 7. 当前缺口

1. PRISMA 2020 的更直接开放原文引文还可继续补
2. 可补 1-2 个关于“AI/机器人领域演示偏差或复现问题”的代表材料，增强第三节说服力
3. 可在后续为本章配一个“整份报告结构图”或“证据类型分层表”
