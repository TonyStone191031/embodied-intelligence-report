# 27 兴趣议题编录与延伸研究 - 研究底稿

## 1. 本章职责

本章不是附录式杂项收容区，而是把那些对长期研究判断有持续牵引力、但又不适合完全摊薄到主线章节里的议题，单独编录成可持续维护的专题入口。

## 2. 当前纳入的两个专题

1. RoboCup：从机器人竞赛到具身系统试验场
2. 多模态大模型编码逻辑：统一 token 接口的能力、边界与潜在替代路线

## 3. 专题纳入标准

一个兴趣议题值得进入本章，至少应满足以下条件之一：

1. 同时牵引多个主线章节的判断框架
2. 近一到两年有持续、公开、可复查的新证据流
3. 能反过来修正整本报告的某条核心判断
4. 适合作为长期论文卡、规则卡或研究问题清单的索引入口

### 4.3 后续版本最值得追的更新问题

1. 规则是否继续把任务复杂度推向动态协同、全场自主和异常恢复
2. 学习方法进入了感知层、局部技能层还是更高层的系统组织层
3. 仿真训练、数据回流和嵌入式部署是否形成更稳定闭环
4. 竞赛内能力是否开始更明确地迁移到家庭服务、移动操作、工业装配或搜救

### 5.4 后续版本最值得追的更新问题

1. 哪些模态必须保留专用结构，不能被过度串行化
2. 哪些能力退化可由更强连接器弥补，哪些退化源于接口定义错误
3. 是否出现更清晰的预算-性能曲线与模态删减实验规范
4. 是否出现更强理论框架来解释“何时应统一、何时应旁路”
5. 在具身系统中，状态/动作旁路是否会逐步变成标配架构

## 6. 本章写作注意事项

1. 本章应避免写成“新闻摘录”或“热点展览馆”
2. 本章每个专题都要对应至少一个主线章节的回调入口
3. 更新时优先补“证据变化 -> 判断变化 -> 是否触发主线回调”
4. 若某个专题长期不再牵引主线，应考虑降级回 `research/` 资产而非继续占据正文专题位
