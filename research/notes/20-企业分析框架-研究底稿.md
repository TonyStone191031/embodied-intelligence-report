# 20 企业分析框架 - 研究底稿

## 1. 本底稿用途

本文件不是正式正文，而是用于后续持续维护企业章节的研究型工作底稿。
目标是把不同企业分析统一到同一问题集下，避免企业专题退化成新闻罗列。

本底稿服务于：

1. `docs/report/current/20-企业分析框架.md`
2. `docs/report/current/21-海外重点企业专题.md`
3. `docs/report/current/22-国内重点企业专题.md`
4. 后续 `data/processed/企业跟踪框架-v0.0.csv` 的字段维护

## 2. 统一分析问题集

每一家企业后续都优先回答以下问题：

1. 本体：控制了什么身体，哪些身体能力是核心壁垒
2. 模型：大模型位于高层接口、中层技能还是低层动作生成
3. 数据：数据来源是什么，是否存在真实部署回流
4. 部署：是否进入真实客户现场，是否有恢复与运维机制
5. 制造：是否具备量产、供应链与成本工程能力
6. 商业模式：卖平台、卖整机、卖方案还是卖服务
7. 风险：当前最脆弱瓶颈是什么

## 3. 企业分析时最容易被误判的点

1. 把高层语义能力误判为完整系统能力
2. 把视频 demo 误判为真实可交付能力
3. 把会造本体误判为已经建立系统护城河
4. 把高融资额误判为技术成熟度领先
5. 把研究影响力误判为现实部署能力

## 4. 后续应持续补充的证据类型

### 4.1 一手或近一手证据

1. 企业官网产品页
2. 官方技术博客
3. 官方论文、白皮书、报告
4. GitHub 仓库与开发文档
5. 官方视频演示与发布会材料

### 4.2 二手但高价值证据

1. 客户案例页面
2. 供应链合作公告
3. 量产、试点、部署相关报道
4. 安全事件、故障回收、监管通报

## 5. 后续可脚本化整理的字段建议

后续做批量资料整理时，可统一采集以下字段：

1. company_name
2. region
3. body_type
4. model_route
5. data_sources
6. deployment_stage
7. manufacturing_signal
8. business_model
9. key_partners
10. key_risks
11. primary_sources
12. updated_at
