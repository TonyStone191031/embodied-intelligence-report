# 08 具身智能系统架构与分层接口 - 研究底稿

## 1. 本章职责

本章是技术主干的总接口章，作用不是重复各个专题技术，而是回答：

1. 现实具身系统为什么几乎总要分层
2. 层边界由哪些时间尺度、信息结构和工程约束决定
3. foundation model 时代，哪些层被重新定义，哪些层并未消失

## 2. 与相邻章节的边界

1. 第 07 章负责解释大模型能提供什么高层能力
2. 本章负责解释这些能力如何嵌入系统分层
3. 第 09~16 章负责把本章各层逐步放大

因此，本章不做算法罗列，而做接口组织。

## 3. 本章核心判断

1. “端到端”叙事并未消除分层，只是把部分接口从显式写死改成隐式学习
2. 架构争论本质上是时间尺度、抽象层次和责任边界的争论
3. 高层统一模型最容易成功的地方通常是任务接口与记忆接口，而不是毫秒级控制
4. 技能层常常是真正的落地中介
5. 安全回退不应是外挂模块，而应是架构内生层

## 4. 本章应重点引用的系统

1. RT-1 / RT-2
2. PaLM-E
3. NVIDIA Isaac / GR00T
4. 行为树、HTN、分层策略等中间表示路线

## 5. 本章应重点解释的接口

1. 感知层输出的不是标签，而是可供后续使用的状态与场景摘要
2. 表征层输出的不是 embedding 本身，而是任务相关内部状态
3. 规划层输出的不是最终连续控制，而是技能目标、阶段目标或结构化子任务
4. 技能层输出的是参数化动作原语、局部轨迹或局部控制参考
5. 控制层负责稳定性、约束满足与安全回退

## 6. 本章应补强的内容

1. 层间信息流、频率差与延迟预算
2. 为什么“统一接口”既重要又危险
3. 为什么大脑-小脑-脊髓不是类比修辞，而是时间尺度组织方式
4. 为什么技能层会成为通用模型和部署现实之间的真实中介

## 7. 本章应出现的公式 / 伪代码

1. 感知输出状态估计形式
2. 潜状态转移与观测预测形式
3. 高层技能选择与技能策略接口形式
4. 计划 -> 技能 -> 失败恢复伪代码
