# v0.3 第 29 章证据底稿：硬件、ROS 2、实时系统与云边端部署

## 主讲边界

- 主讲从工作负载到硬件、通信、ROS 2 运行时和更新回退的部署剖面。
- 控制理论见第 6、7 章；评测统计见第 28 章；安全放行见第 30 章，本章不重复推导。

## 核心来源与可用论断

1. ROS 2 Jazzy QoS 官方文档：DDS 派生的 history、depth、reliability、durability、deadline、lifespan、liveliness；发布者与订阅者策略可能不兼容。
2. ROS 2 Jazzy lifecycle 官方文档：managed node 的 configure/activate/deactivate/cleanup/shutdown 状态与硬件有序启停。
3. ros2_control Jazzy 官方文档（2026-07 抓取）：Controller Manager 执行 read--update--write，Resource Manager 管理 System/Sensor/Actuator 和状态/命令接口。
4. ROS 2 Jazzy security 官方教程：SROS2 构建在 DDS-Security 上，使用 keystore、证书、enclave 和 Enforce 策略；其存在不等于系统自动安全。

## 事实边界

- Jazzy 在 2026-07 仍受支持，但官方页面提示已有更新发行版；正文固定发行版名和访问日期。
- 实时性指截止期可证明/可测地满足，不等于平均运行很快；Python、动态分配和网络路径不应进入硬截止期回路。
- 配置片段为依据官方字段写成的教学配置，不冒充具体机器人仓库原码。
