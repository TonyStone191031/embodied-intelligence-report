# 14 仿真、数字孪生与评测基础设施 - 研究底稿

## 1. 本章职责

本章要把仿真和 benchmark 从“辅助工具”写成训练闭环和部署验证基础设施。

## 2. 核心问题

1. 不同仿真平台各适合什么问题
2. 为什么数字孪生不是“把 3D 场景做真”这么简单
3. sim2real 的失效模式究竟分哪几层
4. benchmark 为什么本身也有方法偏置

## 3. 关键资料

1. Isaac Sim / Isaac Lab
2. Gazebo / ROS 生态
3. MuJoCo
4. PyBullet
5. Habitat
6. BEHAVIOR-1K
7. Domain Randomization

## 4. 本章应补强的论证点

1. 平台选择 = 问题设定选择
2. 观测偏差、动力学失配、接触误差与任务分布偏移需要分层分析
3. 评测要同时看成功率、恢复率、安全性和长时稳定性
