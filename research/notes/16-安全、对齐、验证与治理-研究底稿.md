# 16 安全、对齐、验证与治理 - 研究底稿

## 1. 本章职责

本章要把具身安全从抽象原则写成可分层、可测试、可治理的系统问题。

## 2. 核心问题

1. 为什么具身安全不同于纯软件 AI 风险
2. 物理安全、功能安全、网络安全、模型行为安全如何区分
3. 仿真验证、真机验证、红队测试、人工接管如何形成闭环
4. 安全为什么是商业化前提而不是附属成本

## 3. 关键资料

1. NIST AI RMF
2. ISO 10218
3. ISO 13482
4. 机器人安全、功能安全、AI 治理相关公开资料

## 4. 本章应补强的论证点

1. 责任边界如何反向塑造系统设计
2. 灾难性风险为何比平均成功率更重要
3. 验证闭环必须贯穿生命周期而非单次测试
