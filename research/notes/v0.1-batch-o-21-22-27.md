## 3. 本批核心研究判断

### 3.1 关于 DeepMind 的写法

1. DeepMind 的价值不应只被写成“研究领先”，而应被写成“其接口语言是否正在向执行层下探”。
2. `Gemini Robotics -> Gemini Robotics-ER -> Gemini Robotics On-Device` 这条链，最重要的不是模型名字变化，而是它同时触碰了动作接口、跨本体迁移、端侧部署与安全责任切层。
3. 对 DeepMind 的长期跟踪应固定到五个变量：
   - `interface depth`
   - `embodiment span`
   - `recovery / replanning`
   - `evaluation / SDK openness`
   - `local deployment feasibility`

### 3.2 关于海外第二梯队的写法

1. 第二梯队公司的核心价值，不是补齐名单，而是充当头部叙事的样本外检验。
2. 最值得记录的不是热视频与估值，而是：
   - 任务单元是否稳定
   - 操作员负担是否下降
   - 维护节奏是否收敛
   - 回退规则是否清晰
   - 多站点复制是否开始出现
3. 后续季度更新时，第二梯队应按“问题定义者 / 平台底座构建者 / 本体工程推进者 / 交付组织探索者”四类角色记录，而不应继续追求主观总分。

### 3.3 关于国内企业的写法

1. `104.2-104.4` 的重点不是再讲“国内制造强”，而是把制造与交付能力拆成可验证字段：
   - 出厂一致性
   - 现场标定时间
   - 异常恢复组织
   - 维护节奏
   - 复制能力
2. “自动驾驶/工业机器人能力外溢”必须同时写收益与迁移摩擦，避免把组织履历误写成已完成的系统能力。
3. 成本工程不应只看 BOM，而应把停机、维护、远程诊断、备件网络和测试工装一起算进路线边界。
4. 国内企业更适合用“资源层 -> 能力层 -> 复制层”的三步阅读纪律，而不适合直接沿用海外平台型叙事。

### 3.4 关于兴趣议题章节的写法

1. `27` 章不是附录，而是“主线回调缓冲层”。
2. 一个成熟专题最好同时拥有四层资产：
   - 正文专题
   - 研究底稿
   - 结构化卡片
   - 工作清单中的回调记录
3. 兴趣议题是否触发主线回写，应由“新一手证据 + 影响范围 + 判断变化”共同决定，而不是凭主观兴趣不断加长。
