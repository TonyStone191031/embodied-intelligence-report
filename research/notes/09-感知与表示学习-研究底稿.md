# 09 感知与表示学习 - 研究底稿

## 1. 本章职责

本章要把感知从“CV 子任务集合”重写为“对行动有用的状态构造过程”。

核心目标：

1. 解释 2D、3D、触觉、本体感觉分别提供什么类型的行动约束
2. 解释表示学习如何成为感知和后续规划 / 控制之间的桥梁
3. 解释为什么感知 benchmark 成功不等于可部署感知成功

## 2. 本章与相邻章节边界

1. 第 08 章负责系统分层接口
2. 本章负责感知层与表示层本身
3. 第 10 章负责在表示之上继续做未来预测和世界建模

## 3. 本章应重点回答的问题

1. 为什么机器人感知比通用视觉更强调位姿、遮挡、可供性和时序稳定
2. 为什么第一视角机器人视觉与第三视角互联网视觉存在结构偏移
3. 为什么触觉、力觉和本体感觉在具身系统中不可被边缘化
4. 感知输出究竟应该是什么形式，才能真正被下游系统消费

## 4. 重点资料

1. Segment Anything
2. DINOv2
3. MAE
4. PointNet
5. GelSight
6. 相关三维感知、触觉感知与机器人感知综述

## 5. 本章应补强的论证点

1. 第一视角 / 第三视角差异
2. 触觉与接触感知为何重新升温
3. 弱监督、自监督与任务闭环之间的关系
4. 感知输出不是 embedding，而是可执行状态接口
