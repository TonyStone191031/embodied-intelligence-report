# papers 目录说明

## 用途

`research/papers/` 用于存放论文级长期跟踪材料。

它不替代正文章节，而承担：

1. 代表论文摘录与结构化摘要
2. 论文谱系与时间线补充材料
3. 后续版本更新时的快速回看入口

## 当前建议的使用方式

后续新增论文研究材料时，优先按以下两类文件沉淀：

1. 代表论文卡
   - 适合记录单篇论文的核心结构
   - 包括问题设定、输入输出接口、数据来源、泛化目标、部署假设、局限性

2. 路线型论文清单
   - 适合记录某条主线上的多篇代表工作
   - 例如 VLA、世界模型、端侧轻量路线、shared autonomy

## 建议命名

1. 单篇论文卡：
   - `论文名-论文卡-v0.0.md`

2. 路线清单：
   - `路线名-论文清单-v0.0.md`
