# Code as Policies 论文卡 v0.0

更新时间：2026-07-01

## 1. 基本信息

- 论文名称：Code as Policies: Language Model Programs for Embodied Control
- 年份：2022
- 所属主线：程序化规划 / LLM 代码生成控制
- 对应章节：12、18

## 2. 问题设定

这篇论文要解决的关键问题是：如果语言模型直接输出自由文本，机器人系统难以可靠执行；那能否让模型输出结构更强、可检查、可组合的程序片段，再由程序调用底层感知与控制 API。

## 3. 输入输出接口

- 输入：语言指令、环境上下文、已有 API/函数库
- 输出：可执行代码或程序片段
- 接口类型：语言到程序，再由程序调用机器人技能

## 4. 数据与训练

- 数据来源：语言模型预训练知识 + 程序模板/接口
- 训练方式：代码生成式推理
- 评测方式：任务完成情况、程序可执行性与组合能力

## 5. 论文在谱系中的角色

Code as Policies 代表了另一条与纯 VLA 不同的重要路线：不是让模型直接输出动作，而是先输出结构化可执行中间表示。它对今天很多“程序化代理 + 机器人 API”路线仍有明显启发。

## 6. 工程与部署含义

1. 更适合高层控制、规则约束和人类审查介入。
2. 便于把安全规则、日志和可解释性嵌入执行链。
3. 对复杂任务组合与工具调用尤其有用。

## 7. 局限与后续问题

1. 代码层可执行不等于物理层可执行。
2. 中间表示更清晰，但时延与系统复杂度更高。
3. 需要稳定 API 和技能层作为支撑。

## 8. 后续跟踪点

1. 程序化中间表示是否会在未来 VLA 系统中长期存在。
2. 与 embodied reasoning、外部记忆和安全约束如何结合。
