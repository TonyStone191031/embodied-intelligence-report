# RT-1 论文卡 v0.0

更新时间：2026-07-01

## 1. 基本信息

- 论文名称：RT-1: Robotics Transformer for Real-World Control at Scale
- 年份：2022
- 所属主线：VLA 前史 / Transformer policy / 多任务机器人控制
- 对应章节：07、11、18

## 2. 问题设定

RT-1 试图解决的问题，不是某个单一抓取任务的性能极限，而是“能否把多任务、多环境、自然语言条件下的真实机器人控制，组织成统一的视觉语言动作序列建模问题”。这一步非常关键，因为它把机器人策略学习从任务专用模型推进到更接近 foundation-model-style 的接口。

## 3. 输入输出接口

- 输入：图像观测、自然语言任务描述、历史上下文
- 输出：离散化动作 token
- 接口类型：视觉语言条件下的动作序列建模

RT-1 的关键启发在于，它把复杂连续控制问题部分离散化，并以 token 形式组织动作输出，从而让 Transformer 更自然地进入机器人策略建模流程。

## 4. 数据与训练

- 数据来源：大规模真实机器人数据
- 训练方式：监督式行为克隆/序列建模
- 评测方式：多任务真实机器人控制成功率与泛化能力

## 5. 论文在谱系中的角色

RT-1 的核心推进，不在于首次提出 Transformer 用于机器人，而在于第一次较有说服力地展示：真实机器人控制可以在较大规模多任务数据上被组织成统一模型，并表现出一定泛化能力。它是后续 RT-2、OpenVLA、GR00T 等路线的重要前史节点。

## 6. 工程与部署含义

RT-1 的工程价值主要有三点：

1. 它证明真实世界机器人数据规模一旦上来，统一策略模型会比“每任务一个专用模型”更有吸引力。
2. 它让动作 token 化和多任务条件控制进入主流视野。
3. 它提醒后续路线：数据闭环和统一接口同样重要，不能只盯着模型结构。

## 7. 局限与后续问题

1. 仍然较依赖训练分布，开放世界泛化有限。
2. 语言理解与高层推理能力相对较弱，仍偏“执行型策略模型”。
3. 动作离散化虽然方便序列建模，但也可能压缩控制精度与时序细节。

## 8. 后续跟踪点

1. 它如何被 RT-2、OpenVLA 等后续工作继承与改写。
2. 动作 token 化是否仍是主流接口，还是会被 diffusion/flow matching 等生成式动作建模部分替代。
