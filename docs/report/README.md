# 报告维护说明

## 目标

本目录用于维护《具身智能行业和技术发展调研报告》正文。

## 组织原则

1. 平时按章节拆分维护。
2. 总入口文件负责目录、版本说明、章节索引和整体验证。
3. 图、表、代码片段分别放在 `figures/`、`tables/`、`snippets/`。
4. 最终如需单文件版本或导出版本，统一放到 `output/exports/`。

## 建议写作流程

1. 先在对应章节文件中写正文。
2. 同步把来源笔记沉淀到 `research/notes/`。
3. 图表准备完成后，再回填到章节中。
4. 每完成一个阶段，更新 `CHANGELOG.md`。

## 文件命名

章节文件统一采用：

`NN-章节名.md`

例如：

- `01-报告导读与方法说明.md`
- `02-具身智能的定义、边界与问题框架.md`
