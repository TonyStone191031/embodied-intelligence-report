| --- | --- | --- | --- | --- | --- | --- |
| 填写说明 | 例如 2026Q2 | 融资 / 客户 / 产品 / 论文 / 事故 / 标准 / 供应链 | 用一句话写清“发生了什么，为什么重要” | 数据 / 模型 / 部署 / 制造 / 商业 / 组织 | 上修 / 下修 / 维持 | 是 / 否 |

使用建议：

1. 先分类，再记录，不要直接把新闻抄进正文。
2. 只有当事件改变了原有判断维度时，才需要回调企业专题。
3. 同一季度若仅增加声量、不增加可复用资产，应低权重处理。
