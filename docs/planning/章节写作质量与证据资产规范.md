# 章节写作质量与证据资产规范

## 1. 适用范围与质量基线

本规范把 v0.3 第 6 章“动力学、接触与柔顺控制”和第 23 章“代表性 VLA 的设计比较”已经通过用户评审的写法固化为全书标准。它适用于基础理论章、模型综述章、系统工程章、应用案例章和产业章；不同章型可以调整顺序，但不得删掉与本章核心论断相关的知识链、证据链和实现链。

质量目标不是让每章拥有同样页数，而是让每个核心知识点形成完整闭环：

> 问题与边界 → 定义与符号 → 原理或机制 → 推导或步骤 → 最小算例 → 实现锚点 → 图表证据 → 适用条件与失效 → 小结与评价。

任何只完成“定义—优点—缺点”的知识点，都不算达到本规范。

## 2. 第一性原理决策记录

### 2.1 公理

1. 报告的首要用途是帮助读者建立可计算、可实现、可验证的知识结构；没有服务这一用途的篇幅和资产应删除。
2. 论文图片、代码和文字均可能受独立著作权或许可证约束；“网上可见”“可以下载”和“代码开源”均不自动等于图片可转载。
3. 引用只能解决学术归属问题，不能替代复制、改编和再发布所需的法律授权。
4. 当前报告以个人学习为目的，但仓库、导出 PDF 或未来版本可能被分享；私人使用边界不能被默认为永久发行授权。
5. 任何核心判断必须能回溯到论文、标准、官方代码、实验数据或明确标注的作者推断。

### 2.2 被挑战的假设

| 假设 | 挑战理由 | 公理 | 结论 |
|---|---|---|---|
| 个人学习意味着所有论文原图都可无条件复制 | 私人使用例外不等于对外传播许可 | 2–4 | 修改：仅允许进入受控学习版 |
| 只要写“来源：某论文”就完成合规 | 署名与授权是两个问题 | 2、3 | 否定 |
| arXiv 上的论文都是开放许可 | arXiv 允许作者选择不同许可证，默认非独占分发许可只授权 arXiv | 2、3 | 否定，必须逐版本查看 license 链接 |
| 代码仓库是 Apache/MIT，论文图也自动同许可 | 软件许可证通常不覆盖论文排版和图片 | 2 | 否定，分别核验 |
| 图片越多越像综述 | 无解释、无正文调用的图片只增加噪声 | 1、5 | 否定 |
| 每个公式给出变量表即可 | 变量表不能替代推导、数值意义和失效边界 | 1、5 | 否定 |
| 每个模型单独写一段就是模型综述 | 缺少统一比较轴时只是产品名录 | 1、5 | 否定 |
| 章节模板应完全一致 | 不同章型承担的知识任务不同 | 1 | 修改：统一质量门，不统一机械排版 |

### 2.3 基本事实与推导结论

1. 《中华人民共和国著作权法》第二十四条允许在满足署名、不影响正常使用、不不合理损害权利人利益等条件下，为个人学习、研究或欣赏使用已经发表的作品；同条也规定为说明问题而适当引用、以及教学科研中的少量复制，但后者不得出版发行。
2. CC BY 4.0 明确允许复制、传播和改编，前提是适当署名、链接许可证并说明改动；论文内另有第三方权利标注的元素除外。
3. arXiv 的 perpetual non-exclusive distribution license 只赋予 arXiv 分发权，并明确限制其他主体复用，不能当成 CC 许可。
4. 因此，全书需要同时支持“可再发布的规范正文”和“严格个人使用的学习资产”，二者不能混用同一许可结论。

推导链：事实 1 支持在严格私人范围内制作含原图的学习副本；事实 2 支持将明确 CC BY 的原图进入常规正文；事实 3 排除“所有 arXiv 原图都可自由转载”；事实 4 要求建立双层资产策略。

### 2.4 压力测试

预演失败情景：一年后用户把 PDF 发给同学或上传网盘，而正文没有区分 CC 原图与依赖私人使用例外的原图。即使初始目的仍是学习，传播行为已经改变风险边界。对应措施是：默认正文只收录具有明确再利用许可、已获书面许可或合规改绘的图片；私人学习原图使用独立清单和可替换接口，不作为公开发布物的必要组成。

## 3. 论文原图的分级与使用规则

### 3.1 资产等级

| 等级 | 许可状态 | 常规正文/可分享 PDF | 私人学习版 | 处理要求 |
|---|---|---:|---:|---|
| A | CC0、公共领域 | 可以 | 可以 | 保留来源记录；CC0 仍建议署名 |
| B | CC BY、CC BY-SA | 可以 | 可以 | 署名、许可证链接、改动说明；SA 改编资产遵循相同许可 |
| C | CC BY-NC、CC BY-NC-SA、CC BY-NC-ND | 条件允许 | 可以 | 仅非商业；ND 原则上不得改绘、裁切或叠加标注；逐条核验 |
| D | 已取得权利人书面许可 | 按许可范围 | 按许可范围 | 保存许可文件、用途、期限和署名格式 |
| E | arXiv 非独占分发、传统出版商版权、许可不明 | 不直接复制 | 可在严格私人学习范围审慎使用 | 常规正文改绘或申请许可；不得把 arXiv 可下载误写成开放许可 |
| F | 来源不明、权属冲突、论文注明第三方素材 | 不使用 | 原则上不使用 | 先找到原始权利人和许可 |

### 3.2 两种构建口径

1. **规范构建（默认）**：允许 A–D 级资产和合规改绘图，可用于正常评审、分享和未来发布。
2. **私人学习构建（可选）**：可以在严格不传播的前提下加入少量 E 级论文原图，用于逐图研读；图片必须标记 `private-study-only`，不得成为公开构建的硬依赖。
3. 私人学习构建不得上传公共仓库、公开网盘或公开网页，也不得自动进入 frozen release；若未来需要分享，先替换为许可原图或改绘图。
4. 无论哪种构建，都要注明作者、论文名、图号、版本、页码、URL/DOI 和使用方式。

### 3.3 原图进入正文的最低判断程序

每幅候选图依次回答：

1. 图是否来自论文作者本人，还是论文再次引用的第三方素材？
2. 核验的是哪一个版本：期刊版本、会议版本还是 arXiv 版本？不同版本可能采用不同许可。
3. 页面上的 license 链接具体指向 CC BY、CC BY-NC-ND，还是 arXiv 非独占分发许可？
4. 许可证是否覆盖图片，图片旁是否另有版权声明？
5. 是否进行裁切、翻译、加框、重配色或拼接？这些行为是否构成改编？
6. 该图是否被正文至少一次明确调用，并承担不可被一段文字或简单自绘图替代的解释任务？
7. 资产登记是否完整，公开构建与私人构建边界是否明确？

任一权属问题无法回答时，资产降为 E/F 级，不得凭“非商业”猜测许可。

### 3.4 图注模板

直接复用 CC BY 原图：

> 图 X：……。原图引自 Author et al., “Paper Title,” Fig. N，arXiv/version 或 DOI，CC BY 4.0；未修改/仅缩放。

改绘图：

> 图 X：……。作者依据 Author et al. 原论文 Fig. N 改绘；结构关系保持一致，配色、标注与布局已重制，不复制原图像素。

私人学习原图：

> 图 X：……。原图引自……，仅收入私人学习构建，不授权再传播；公开构建使用图 Y 的改绘版本。

## 4. 全书统一的知识点写作单元

每个核心知识点不要求单独设小节，但正文必须覆盖以下字段：

1. **要解决的问题**：先说明输入、输出、任务边界和为什么现有方法不足。
2. **定义与符号**：变量、维度、单位、坐标系、随机变量或张量形状。
3. **机制**：用因果链、数据流或状态转移解释“它如何工作”。
4. **数学或算法**：给出关键公式链、优化目标、更新规则或伪代码。
5. **最小算例**：至少进行一次代数代入、数值计算、张量形状传播或时序展开。
6. **实现接口**：真实源码、官方 API、文件路径、符号名、tag/commit 和许可证；无可用源码时明确说明。
7. **证据资产**：论文原图/改绘图、实验表或数据图至少一种，并在正文解释读图结论。
8. **适用与失效**：列出假设被破坏时的现象、诊断信号和替代方案。
9. **文献位置**：说明原始工作、代表性改进和仍有争议的结果。
10. **本节结论**：只总结本知识点产生的新理解，不重复背景口号。

一个知识点如果缺少第 4–8 项中的三项以上，必须合并到其他知识点或补充研究，不能靠增加评价段落掩盖。

## 5. 公式讲解协议

每个核心公式至少回答八个问题：

1. 公式从哪个问题或物理/概率假设出发？
2. 每个符号的维度、单位和坐标系是什么？
3. 等式或目标函数是定义、近似、约束，还是可证明结论？
4. 中间推导最少需要哪几步？不得只写“可得”。
5. 用一组数字或张量尺寸代入后会得到什么？
6. 参数增大或减小时，结果如何变化？
7. 哪些条件下逆矩阵不存在、梯度失效、近似偏离或数值不稳定？
8. 公式在真实代码中对应哪一段计算？

公式不得孤立陈列。连续出现两个以上核心公式时，应使用承接句解释前式输出如何成为后式输入。

## 6. 模型讲解协议

每个核心模型采用统一的十一字段卡片，但写入自然正文而不是机械填表：

1. 问题设定与设计动机；
2. 输入组织、模态和历史窗口；
3. tokenizer/encoder/connector；
4. 主干结构与信息流；
5. 输出头、动作表示或预测变量；
6. 训练目标和关键损失；
7. 数据组成、规模、清洗和归一化；
8. 训练循环与推理流程；
9. 闭环频率、动作块、缓存和低层接口；
10. 官方源码关键路径、权重和复现条件；
11. 实验、消融、局限和证据边界。

模型比较不得按模型名顺次写宣传式简介。先确定统一比较轴，再把模型作为不同设计选择的证据。第 23 章采用的有效比较轴包括动作表示、主干/动作头分工、训练目标、动作块和闭环方式。

## 7. 不同章节类型的组织方式

### 7.1 基础理论章

学习目标与前置知识 → 问题边界 → 推导主线 → 数值算例 → 算法/源码 → 工程失效 → 文献脉络 → 知识小结 → 综述判断。

基础章至少包含一条完整推导、一个可复算例、一幅原理图、一段真实实现或明确教学伪代码，以及一次工程失效分析。

### 7.2 模型综述章

统一问题设定 → 设计空间 → 各路线机制 → 代表模型拆解 → 源码路径 → 实验与消融 → 横向表 → 局限与适用条件 → 综述判断。

模型综述章至少包含一个统一比较框架；核心模型各自满足第 6 节字段，不允许只给参数量、数据量和成功率。

### 7.3 系统工程章

系统目标 → 组件职责 → 接口/数据流 → 时间尺度 → 部署拓扑 → 监控与恢复 → 完整案例 → 检查表。

系统章必须给出正常路径和失败路径，说明每个接口的数据类型、更新频率、延迟预算和责任边界。

### 7.4 应用案例章

任务定义 → 场景约束 → 系统栈 → 数据与训练 → 部署流程 → 指标 → 成本 → 失败样本 → 可迁移结论。

应用章不得汇编场景名录；每个案例必须有可核系统证据，并说明演示、试点和规模部署的区别。

### 7.5 产业与政策章

可核事实 → 统一证据矩阵 → 技术/交付/制造/运维比较 → 不确定性 → 作者判断。

企业宣传、融资新闻和政策口号只能支持事件事实，不能直接支持技术成熟度或商业可行性结论。

## 8. 图、表、代码与正文的耦合规则

1. 每幅图必须在正文中以“图 X 展示……，其中……”方式解释至少一个具体关系。
2. 原理图解释机制；实验图支持性能或消融论断；系统图表达接口和时间尺度；实物图只在形态本身影响理解时使用。
3. 表格必须有统一比较字段，空泛的“优点/缺点”表不得代替正文分析。
4. 合成教学图必须标注“教学示意/合成数据”，不得让读者误认为论文实验。
5. 源码片段以 10–30 行为宜，保留决定算法行为的路径；同时解释输入、输出、状态和与公式的差异。
6. 图、表和代码不是计数目标。若正文不调用、不能回答问题或没有来源记录，应删除。

## 9. 语言与段落协议

1. 先写事实和机制，再写评价；作者评价使用明确标识。
2. 删除“值得注意的是”“不容忽视”“具有重要意义”等没有新增信息的起句。
3. 每段承担一个论证动作：定义、推导、算例、证据、反例或结论。
4. 不为不足两个实质段落的内容单设标题；避免标题碎片化。
5. 章首不写版本维护和阅读说明；章末只保留一次知识小结和一次综述判断。
6. 使用自然中文解释专业概念，英文术语只用于首次定义、标准名称、代码符号和检索需要。

## 10. 章节级验收表

### 10.1 内容闭环

- [ ] 学习目标与前置知识明确。
- [ ] 核心知识点均有唯一主讲位置。
- [ ] 核心公式完成假设、变量、推导、算例和失效解释。
- [ ] 核心模型完成十一字段拆解。
- [ ] 至少一个真实实现或明确标注的教学伪代码。
- [ ] 知识讲解与作者评价分开。

### 10.2 证据与资产

- [ ] 学术事实均有正文引用。
- [ ] 数值、SOTA、商业部署和政策结论逐项可追溯。
- [ ] 每幅图有来源、许可、图号/页码和处理方式。
- [ ] 每段源码有仓库、许可证、tag/commit、文件路径和符号名。
- [ ] 教学图、论文实验图和公开照片没有混淆。
- [ ] 图表均被正文解释，且不是装饰性资产。

### 10.3 可读性与构建

- [ ] 无连续堆公式而缺少承接解释。
- [ ] 无模型名录、公司名录和场景名录。
- [ ] 无过短小节和重复总结。
- [ ] LaTeX/Biber 构建成功，未定义引用和缺失资产为 0。
- [ ] 抽样检查公式、长表、代码、图片和 References 页面。
- [ ] 私人学习资产未进入可分享构建。

任一“核心公式”“核心模型”“图片授权”项未通过时，该章不得标为 completed。
