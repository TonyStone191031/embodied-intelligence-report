# V0.0 批次执行计划

## 1. 目的

本计划是《具身智能行业和技术发展调研报告-v0.0》的低打扰执行面板。

它不再回答“这本报告大概要写什么”，而是回答：

1. 目前每一章还差什么
2. 每一章下一步的明确目标是什么
3. 每一章需要并行完成的两类工作是什么
4. 什么时候可以认为该章暂时达到了当前轮次的退出条件

本计划与以下文件配套使用：

1. `docs/planning/项目工作规范.md`
2. `docs/planning/具身智能报告-V0.0-详细大纲.md`
3. `docs/planning/V0.0-低打扰自主推进方案.md`

---

## 2. 当前默认执行模式

从当前阶段开始，正文推进采用“章节级目标 + 待完成清单 + 批次收口”的方式，而不是“写一章问一次用户”。

对每一章默认并行推进两类工作：

1. 正文精修
2. 结构化沉淀

其中：

1. “正文精修”指补厚论证、补引用、补公式、补伪代码、补图表位、补案例与工程约束
2. “结构化沉淀”指把该章的共性资料沉淀到 `research/`、`data/`、`assets/`、`docs/report/current/tables/` 等长期可复用位置

---

## 3. 本轮优先级规则

### 3.1 优先处理对象

优先处理满足以下任一条件的章节：

1. 三级标题下仍存在一两句话的短段
2. 技术主干章节中公式、伪代码、系统图明显不足
3. 产业与企业章节中案例密度、横向比较或引用密度不足
4. 已经形成框架，但还没有沉淀对应研究底稿或结构化表格
