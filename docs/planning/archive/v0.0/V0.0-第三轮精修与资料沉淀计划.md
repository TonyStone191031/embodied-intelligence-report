# V0.0 第三轮精修与资料沉淀计划

## 1. 目的

本计划用于把《具身智能行业和技术发展调研报告-v0.0》从“完整可读初稿”推进到“结构化素材更齐全、可持续维护性更强的研究型工作版本”。

它回答的不是“还要不要继续写正文”，而是：

1. 哪些工作应优先从长文本中抽离为结构化资产
2. 哪些图表已经具备脚本化或半脚本化生成条件
3. 哪些章节应进入第三轮精修
4. 本轮结束后，什么状态才算真正进入下一阶段

本计划与以下文件配套使用：

1. `docs/planning/项目工作规范.md`
2. `docs/planning/具身智能报告-V0.0-详细大纲.md`
3. `docs/planning/V0.0-低打扰自主推进方案.md`
4. `docs/planning/V0.0-批次执行计划.md`

---

## 2. 当前阶段判断

截至当前版本，`docs/report/current/` 已形成按章节拆分的完整初稿，主要技术主干与产业章节均已具备可连续阅读能力。

但如果继续仅靠“在正文里补更多段落”推进，后续会出现三个问题：

1. 共性判断框架重复散落在不同章节中，不利于版本更新
2. 图表虽然已有第一批 mermaid 与表格，但尚未形成统一资产清单
3. 一些章节虽然已经不薄，但仍缺少足够稳固的资料底座来支撑后续季度更新

因此，本轮默认切换为“正文第三轮精修 + 结构化资产沉淀 + 脚本化制图准备”三线并行。

---

## 3. 本轮三条主线

### 3.1 主线 A：结构化资料沉淀

目标：

把已经在正文里反复出现、并会在后续版本中持续复用的分析框架，优先沉淀到 `research/`、`data/processed/` 和 `docs/report/current/tables/`。

本轮优先沉淀对象：

1. 图表资产清单
2. 论文谱系时间线数据源
3. 企业季度变更与正文回调索引
4. 章节间共用的术语、判断框架与复审口径

### 3.2 主线 B：图表体系与脚本化准备

目标：

把现有 mermaid 图、表格和后续待补图表整理成统一资产体系，为下一步进入 `code/` 下的批量制图脚本准备底层清单。

本轮优先动作：

1. 建立图表资产总清单
2. 标注每张图当前状态
3. 标注每张图对应章节与源文件路径
4. 区分“已落正文”“已有源文件未系统复核”“后续应补第二批图”

### 3.3 主线 C：第三轮正文精修

目标：

不再广撒网补字，而是优先处理那些已经有骨架、但仍需要更高引用密度、更多工程接口说明、更多结构化支撑的小节与章节。

本轮优先章群：

1. `12-规划、推理与决策的新路线.md`
2. `13-数据工程与训练闭环.md`
3. `14-仿真、数字孪生与评测基础设施.md`
4. `16-安全、对齐、验证与治理.md`
5. `17-学术前沿技术专题.md`
6. `19-开源生态、代码库与工具链.md`
7. `21-海外重点企业专题.md`
8. `22-国内重点企业专题.md`
9. `23-应用落地与商业化.md`
10. `26-结论与后续跟踪清单.md`

## 4. 本轮优先顺序

### 4.1 第一阶段：把“资产底座”补出来

优先完成：

1. 图表资产清单
2. 数据清单与用途说明
3. research 目录说明与资料落点说明
4. 当前版本状态说明更新

退出条件：

1. 不再依赖会话记忆判断“哪些图已经有源文件”
2. 不再依赖人工回忆判断“哪些结构化表已经建好”

### 4.2 第二阶段：围绕清单回补正文

优先完成：

1. 把相关表格、图表、说明文字进一步挂回正文
2. 把仍然偏“短注释”的小节补到正式正文强度
3. 把需要长期复用的判断口径同步回写到 `research/notes/`

退出条件：

1. 正文与结构化资产之间形成稳定引用关系
2. 重点章节不再主要依靠口头性总结支撑判断

### 4.3 第三阶段：脚本化制图准备

优先完成：

1. 判断哪些图适合继续保留 mermaid
2. 判断哪些图需要转为脚本生成表图
3. 为 `code/` 下后续脚本化生成准备输入清单

退出条件：

1. 已能明确下一步该在 `code/` 里生成什么
2. 图表生成对象、输入源和输出目录关系已明确

---

## 5. 目录动作要求

### 5.1 `research/`

本轮重点不是塞更多零散资料，而是补齐“说明文件 + 共性底稿 + 可复用索引”。

优先动作：

1. 建立或更新 `README`
2. 补充跨章节底稿
3. 说明 `articles/`、`papers/`、`companies/`、`datasets/`、`timelines/` 的后续用途

### 5.2 `data/processed/`

本轮重点是：

1. 明确每个 `.csv` 的字段用途
2. 明确它服务哪一章或哪一类更新工作
3. 为后续脚本输入做准备

### 5.3 `assets/diagrams/`

本轮重点是：

1. 建立图表源文件清单
2. 标注对应章节
3. 标注正文挂载状态
4. 标注是否需要第二轮重绘

### 5.4 `docs/report/current/`

本轮重点不是大改结构，而是：

1. 保持章级文件结构稳定
2. 将补充后的结构化资产尽量接回正文
3. 持续提高章节的研究型强度

---

## 6. 复审标准

本轮结束前，至少应检查：

1. 当前正文是否已经有清晰的资产配套
2. 当前数据表是否已经可作为脚本输入雏形
3. research 是否已经开始承担长期记忆职责
4. 重点章节是否仍有明显的一两句短段
5. 图、表、正文之间是否已形成稳定的编号与路径关系

---

## 7. 本轮完成标志

当满足以下条件时，可认为本项目已正式进入“批量资料整理、脚本化图表生成”阶段：

1. `research/`、`data/processed/`、`assets/diagrams/` 都已有可读的索引或说明文件
2. 结构化资产清单已能覆盖当前第一批核心图表与表格
3. 正文中关键章节已完成第三轮精修的第一批回补
4. 可以明确列出下一步应在 `code/` 中实现的脚本化制图任务
