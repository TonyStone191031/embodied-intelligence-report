# 《具身智能行业和技术发展调研报告-v0.0》准备方案

## 1. 本阶段目标

本阶段只完成三件事：

1. 为长期调研安装和确认必要的 Codex skills。
2. 设计可长期维护的项目目录结构与资料组织规范。
3. 给出一份适合 100 页以上系统调研报告的详细大纲，供后续正式写作使用。

本阶段不开始撰写正式报告正文。

## 2. 已安装与可用的 skills

### 2.1 本次新安装

1. `screenshot`
   - 用途：快速保存网页、图表、公告页截图，便于报告插图留档。
2. `jupyter-notebook`
   - 用途：做研究笔记、数据整理、企业清单统计、时间线分析。
3. `pdf`
   - 用途：处理白皮书、论文 PDF、导出的报告 PDF 与版式核查。
4. `playwright`
   - 用途：自动化抓取官网、博客、GitHub 页面中的结构化信息与截图。

### 2.2 会直接配合使用的内置能力

1. `web`
   - 用途：检索最新论文、官网新闻、行业动态与引用链接。
2. `browser:control-in-app-browser`
   - 用途：后续如需对网页可视化内容做精细检查，可配合浏览器操作。
3. `documents / presentations / spreadsheets`
   - 用途：后续如果需要导出 docx、pptx、xlsx，可直接复用。

### 2.3 对 “deepresearch” 的处理结论

本次核查了官方 skill 安装仓库的可安装列表。截至 2026-06-27，官方可列出的 curated skills 中未发现名为 `deepresearch` 的技能包，也未发现可直接访问的 `skills/.experimental` 目录。因此本项目当前采用“已安装 skills + 联网检索 + 项目内规范化沉淀”的方式替代泛化的 deep research 能力。

```text

  scrapers/           # 抓取脚本

output/
  exports/            # 导出产物，例如 pdf/docx/html
```

## 4. 资料组织规范

### 4.1 命名规则

1. 论文：`年份-机构-标题缩写.md`
2. 企业：`企业名-更新日期.md`
3. 事件时间线：`timeline-主题.md`
4. 截图：`日期-来源域名-主题.png`
5. 图表：`章节号-主题名.ext`

### 4.2 引用与证据规则

1. 学术观点优先引用论文、项目主页、官方技术报告。
2. 产品与企业动态优先引用官网、博客、GitHub、官方演讲材料。
3. 行业评价与市场数据单独标记为“行业观点”或“媒体/研究机构观点”。
4. 每一节尽量保留“事实、解释、评价”三层分离，避免把判断写成事实。

### 4.3 版本迭代规则

1. 报告文件名采用 `《具身智能行业和技术发展调研报告-vX.Y》`。
2. `v0.0` 目标是完成全景结构与初版内容，不追求所有企业和论文都穷尽。
3. 后续可按月或按重大里程碑事件升级，例如 `v0.1`、`v0.2`。
4. 每次版本升级单独维护更新日志，注明新增论文、企业、产品、政策和观点修订。

## 5. 对现有参考提纲的批判性判断

### 5.1 可取之处

1. 已经覆盖了宏观、基础算法、工程系统、前沿模型、企业、应用、治理等大块。
2. 明确强调了 VLA、世界模型、JEPA、大小脑等当前主流话题。
3. 有较强的“产业+技术”混合视角，适合做个人长期学习材料。

### 5.2 主要不足

1. 章节重心偏“热词枚举”，但对“机器人学基础”和“评价框架”还不够系统。
2. 对机器人本体类型、任务类型、控制层级、数据来源等分类体系不够清楚。
3. 对 benchmark、评测指标、数据闭环、sim2real、部署约束、安全验证的覆盖不足。
4. 企业章节容易膨胀成新闻罗列，缺少统一分析框架。
5. 大模型兴起后的变化写到了，但“大模型之前的机器人学主脉络”仍不够扎实。
6. 近两个月的新进展尚未被纳入，例如安全基础设施、on-device 路线、VLA 细分场景深化、商业化与资本市场新动态。
