## 启动原因

v0.2 已建立 LaTeX 主源和可复现 PDF 链路，但正文质量不能通过继续增补局部段落解决。2026-07-13 的只读审计得到以下基线：

1. 全书约 58.8 万字符，拆成 27 个部分、122 个数字章节和约 602 个标题单元，知识颗粒过碎。
2. 正文没有正式 `\\cite` 引用，引用库只有 2 条记录，约 458 个 URL 主要以正文链接替代论文引用。
3. 全书约 26 幅图、25 张表和 99 个代码环境；图表以自绘概念图为主，代码多为无版本锚点的教学骨架。
4. 基础部分经常停留在定义、单个公式和评价，缺少推导、算例、真实实现、实验结果与失败条件。
5. 开头、结尾和多个“为什么重要”小节重复承担判断与版本说明，削弱技术主线并放大模板化语气。

因此，v0.3 将 v0.2 视为素材库和冻结基线，不采用逐段补丁式扩写。

## 第一性约束

1. 教材型综述必须让读者从定义和数学原理走到算法、实现与工程边界。
2. 文献综述中的事实、实验和判断必须能回溯到论文、标准、官方代码或正式文档。
3. 每个概念只设一个主讲位置；其他章节通过交叉引用调用，不重复解释。
4. 评价位于原理、实现和证据之后，并与知识讲解显式区分。
5. 篇幅由完整知识链决定，不靠增加标题、重复总结或背景议论堆积。

## 本轮目标

1. 将 27 部分重构为 10 个主题部分、38 个章节、独立 References 与 6 个附录。

6. 删除版本维护教程、重复“为什么重要”、空泛结论和无证据的趋势判断。

## 统一章节写作协议

### 基础理论章

学习目标与前置知识 → 问题定义 → 数学推导 → 算例 → 算法/实现 → 工程失效 → 文献脉络 → 知识小结 → 综述判断。

### 模型综述章

问题设定 → 架构与训练目标 → 数据和训练流程 → 官方源码关键路径 → 实验与消融 → 横向比较 → 局限与适用条件 → 综述小结。

### 系统工程章

接口和数据流 → 组件职责 → 时间尺度与部署 → 监控和恢复 → 完整案例 → 检查表。

### 产业章

可核事实 → 部署证据 → 统一指标比较 → 不确定性 → 作者判断。企业宣传和融资新闻不得直接作为技术结论。

## 文献、图表与代码口径

1. 正文学术事实使用 `\\cite`；References 只列正文实际引用条目，删除 `\\nocite{*}`。
2. 来源优先级：原始论文/标准 → 官方代码与技术文档 → 高质量综述 → 行业统计 → 媒体或企业材料。
3. 每个核心模型建立“原始论文 + 官方仓库/文档 + 复现或后续评测”证据包。
4. 每个核心技术章原则上包含一幅原理图、一张比较或实验表、一个公式链和一段真实代码或明确标注的伪代码。
5. 论文图优先使用允许再利用的版本；否则改绘并记录原论文、图号、许可证和资产路径。
6. 源码片段记录仓库、许可证、tag/commit、文件路径和符号名；教学伪代码不得冒充真实源码。
