# v0.3 证据与资产规划

## 1. 目的

本文件把“需要更多文献、图、表、公式和代码”转化为可执行的章节级证据任务。它不是数量 KPI；任何资产必须直接服务于章节学习目标或论断验证。

## 2. 证据优先级与引用实现

1. 原始论文、正式标准和规范。
2. 官方代码仓库、版本化技术文档和数据集说明。
3. 高质量综述、教科书和课程资料，用于建立范围和教学解释。
4. 官方统计、认证、客户案例和监管材料。
5. 企业博客、新闻稿和媒体，只支持事件事实或企业自述，不支持独立技术结论。

正文采用 GB/T 7714—2015 数字制。工具链恢复后，将 `biblatex` 样式切换为 `gb7714-2015` 并确认本机宏包可用；删除 `\\nocite{*}`。每个引用条目至少包含作者/机构、标题、年份、载体、DOI/URL 和访问日期（仅网页）。

## 3. 章节论断矩阵字段

每章研究底稿必须维护以下字段：

| 字段 | 内容 |
|---|---|
| Claim ID | 例如 `C06-012` |
| 论断 | 可被引用或证伪的完整句子 |
| 类型 | 事实 / 推断 / 评价 |
| 主来源 | 原始论文、标准或官方材料 |
| 交叉来源 | 独立复现、综述或对照材料 |
| 证据位置 | 页码、图号、表号、章节或代码符号 |
| 适用边界 | 平台、任务、数据、时间与实验条件 |
| 正文位置 | 章节和小节 |
| 状态 | 待核验 / 已核验 / 有争议 / 不采用 |

## 4. 图表与代码资产字段

### 图和论文原图

- Asset ID、标题、服务的问题、原论文/网页、页码或图号、许可证、原图/改绘、源文件、本地输出、正文锚点。
- 公开可见不等于允许转载；无法确认授权时只改绘核心关系，并明确“改绘自”。

### 表格

- Table ID、比较对象、字段定义、数值单位、数据日期、逐行来源、缺失值规则、不可比条件和维护人/批次。

### 源码片段

- Snippet ID、仓库、许可证、tag/commit、文件路径、符号名、行段、依赖版本、是否实际运行、正文解释和本地镜像路径。
- 真实源码优先截取 10–30 行关键逻辑；更长实现改为流程图和仓库索引。
- 教学伪代码必须标注“教学化伪代码”，不得使用虚构 API 暗示可运行。

## 6. 数量与密度目标

1. 冻结目标约 200 条以上实际引用的独立来源，多数为论文、标准、官方文档和代码仓库。
2. 预计 50–70 幅有效图、30–45 张比较/实验表、25–40 个关键代码片段。
3. 这些范围用于发现内容过薄，不是强制填充；同一资产不得跨章重复占位。
4. 每个核心技术章至少有一项可计算推导、一项实现证据和一项实验/工程证据。

## 7. 资料落盘位置

- 章节研究底稿：`research/notes/v0.3-chXX-*.md`
- 论文卡：`research/papers/`
- 企业证据卡：`research/companies/`
- 可复用图源：`assets/images/`、`assets/diagrams/`
- 当前版渲染图：`docs/report/latex/current/figures/`
- 当前版代码摘录：`docs/report/latex/current/snippets/`
- 引用库：`docs/report/latex/current/references/references.bib`
- 冻结时随完整 LaTeX 工作区复制到 `docs/report/versions/v0.3/latex/`

## 8. 章节研究放行门

章节从 `planned` 转为 `in_progress` 前必须满足：

1. 学习目标、前置知识和主讲边界已确定。
2. 核心论断矩阵已建立，至少定位原始论文或官方来源。
3. 图、表、公式和代码各自是否需要已经作出决定；不需要时写明理由。
4. 论文图的授权/改绘路线和源码许可证可核查。
5. 与其他章节的重复点已转成明确交叉引用。
