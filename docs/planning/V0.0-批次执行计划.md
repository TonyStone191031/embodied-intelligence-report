# V0.0 批次执行计划

## 1. 目标

本计划服务于《具身智能行业和技术发展调研报告-v0.0》的低打扰自主推进。

它回答的不是“整份报告写什么”，而是“接下来按什么批次、什么顺序、什么质量门来连续写完剩余章节初稿”。

## 3. 批次级质量门

每个批次结束前至少检查：

1. 本批次章节是否完成研究底稿
2. 正文是否已达到“研究型正文初稿”而非概述稿
3. 是否为每章补入了足够的一手来源
4. 是否至少为关键章节预留了图、表、公式或代码位
5. 相邻章节间是否职责清楚、接口明确

---

## 4. 执行顺序决策

从当前时点起，按以下顺序推进：

1. 先补强 `05`

3. 再完成 `07`
4. 再完成 `08`
5. 基础层收口
