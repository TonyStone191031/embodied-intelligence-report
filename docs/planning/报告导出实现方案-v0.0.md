# 报告导出实现方案-v0.0

## 1. 目标

为《具身智能行业和技术发展调研报告》建立一套可重复执行、可版本化维护的导出链路：

1. 源文件继续保持为分章 Markdown。
2. 在 `current/` 或 `versions/<version>/` 的基础上生成整书合并稿。
3. 由同一份合并稿继续导出 Word 与 PDF。
4. 导出产物统一落到 `output/exports/<version>/`。
5. 默认以 Word 作为主要可读交付稿，PDF 作为辅助发布/打印产物。

## 2. 第一版采用路线

第一版采用：

`Markdown bundle -> DOCX -> PDF`

考虑到当前项目是在 Windows 本机长期维护，并且本机已具备 Microsoft Word，因此实现策略调整为：

1. `build_report_bundle.py` 负责按章节顺序合并 Markdown，并补充配套表格附录。
2. `export_docx.py` 直接从合并后的 Markdown 生成 `.docx`。
3. `export_pdf.py` 优先从生成好的 `.docx` 再导出 `.pdf`；只有在 Word 自动化不可用时，才回退为从同一份合并后的 Markdown 直接生成 `.pdf`。
4. `run_export.py` 负责串联完整流程。

这意味着第一版的 Word 和 PDF 仍共享同一份语义源，但 PDF 的首选生成路径改为“继承 Word 版式”。
在当前实现原则下，应优先保证 Word 的目录、公式、图表和正文排版质量；PDF 则优先继承 Word 的分页与样式，当环境能力不足时再回退为辅助性的脚本直出件。

## 3. 第一版能力边界

第一版优先保证“稳定可用”和“长期可维护”，而不是一开始就追求最复杂排版。

优先支持：

1. 标题层级
2. 普通段落
3. 无序/有序列表
4. fenced code block
5. 数学公式块的原样保留
6. Markdown 管道表格
7. 本地 Markdown 表格附录合并
8. 基本页眉页脚与目录性元信息

暂不追求：

1. Mermaid 自动渲染为矢量图
2. 所有 Markdown 内联样式的完整保真
3. 复杂交叉引用的自动编号回填
4. 复杂数学排版完全等价于专业公式编辑器

## 4. 目录与产物约定

脚本目录：

1. `code/export/common.py`
2. `code/export/build_report_bundle.py`
3. `code/export/export_docx.py`
4. `code/export/export_pdf.py`
5. `code/export/run_export.py`

导出目录：

1. `output/exports/<version>/report-bundle.md`
2. `output/exports/<version>/report-bundle.manifest.json`
3. `output/exports/<version>/embodied-intelligence-report-<version>.docx`
4. `output/exports/<version>/embodied-intelligence-report-<version>.pdf`

## 5. 执行方式

建议统一通过总控脚本调用：

```bash
python code/export/run_export.py --version v0.0 --source frozen
python code/export/run_export.py --version v0.0 --source current
```

其中：

1. `--source current`：从 `docs/report/current/` 导出，但产物仍落到 `output/exports/<version>/`
2. `--source frozen`：从 `docs/report/versions/<version>/` 导出

## 6. 后续增强方向

第一版稳定后，后续可继续增强：

1. Mermaid 渲染为图片并回嵌
2. 版本专用 `figures/` 与 `snippets/` 自动附录化
3. 更完整的内联链接、脚注、图表标题与交叉引用支持
4. 增加 DOCX 渲染校验截图链路，进一步自动化检查公式、表格和流程图排版
5. 若后续迁移到无 Word 环境，再补充 `soffice` 或其他无桌面依赖的 PDF 备选链路
