# code 目录说明

## 用途

`code/` 用于存放本项目内部的辅助脚本，包括但不限于：

1. 图表资产校验
2. 结构化数据检查
3. 批量制图
4. 报告导出与整理辅助工具

建议将报告导出链路统一放在 `code/export/`，例如：

1. `build_report_bundle.py`：按章节顺序合并 Markdown
2. `export_docx.py`：导出 Word
3. `export_pdf.py`：导出 PDF
4. `run_export.py`：一键执行完整导出流程

## 当前阶段

当前阶段的重点不是一次性搭建复杂工具链，而是先建立最小可复用脚本：

1. 读取 `data/processed/` 中的结构化清单
2. 校验对应资产路径是否存在
3. 生成可读的检查结果
4. 把部分结构化 `.csv` 自动导出为正式 markdown 表
5. 为版本冻结后的报告导出提供稳定脚本入口

后续如果进入批量制图阶段，应优先在本目录扩展，而不是把临时脚本散落到其他路径。
